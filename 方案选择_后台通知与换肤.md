# 方案选择：后台通知与换肤

## 思路

Android 后台通知由客户端接收 FCM 数据消息后，通过 Notifee 立即创建本地系统通知。换肤由 Redux 持久化主题值，并在 React 组件内通过 `useTheme` Hook 获取颜色。

## 优点

- 不需要改动 Chatwoot 服务端即可修复数据消息不显示的问题。
- Hook 在 TypeScript 编译为 JavaScript 后仍在运行时生效。
- 主题值退出应用后仍保留，且不依赖可变的全局 Tailwind 配置。

## 缺点

- 首轮仅覆盖设置页及系统栏；其他页面需要逐步改用语义颜色令牌。
- FCM 原生通知载荷由 Android 系统显示，客户端不重复创建本地通知。

## 适用场景

- Android 使用数据型 FCM 推送且需要后台及时显示通知。
- React Native 项目需要先建立轻量、可扩展的手动明暗主题基础。
