# 方案选择：后台通知与换肤

## 思路

Android 通知由存活的 ActionCable 连接接收 `notification.created` 事件后，通过 Notifee 创建本地系统通知。换肤由 Redux 持久化主题值，并在 React 组件内通过 `useTheme` Hook 获取颜色。

## 优点

- 不需要 Google Play 服务、Firebase 或厂商推送配置。
- Hook 在 TypeScript 编译为 JavaScript 后仍在运行时生效。
- 主题值退出应用后仍保留，且不依赖可变的全局 Tailwind 配置。

## 缺点

- 首轮仅覆盖设置页及系统栏；其他页面需要逐步改用语义颜色令牌。
- Android 可能在休眠、长时间后台或杀进程后关闭连接，通知不保证送达。

## 适用场景

- 不要求进程被杀后仍通知的 Android 实时会话场景。
- React Native 项目需要先建立轻量、可扩展的手动明暗主题基础。
